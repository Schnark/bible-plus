Usage

1. Basic Usage

This app is a reader mainly intended for bible translations, but also for other texts like encyclicals or any other documents.

All your texts form a library, which consists of collections, collections consist of books, books consist of chapters. For bibles, the collection is a bible translation, and each biblical book is a book in that collection.

The start screen allows you to select a collection, then you select the book in that collection, and finally the chapter you want to start reading with.

The header of the main screen shows the current collection (which allows you to quickly switch to the same book in a different collection, i.e. in a different translation; you can also initiate a side-by-side view comparing two versions) and the current book (which allows you to quickly switch to a different chapter or different book). You can access the history to return to a book you previously read, start a search (in the current book, the current collection or the whole library), or access your bookmarks.

To add a bookmark just click or tap the verse number (or the bookmark icon, if there are no verse numbers). You can also add a note and a highlight color. Note that this help is special because it doesn’t have verses or anything similar, so you cannot add bookmarks here.

To view a bookmark just click or tap the bookmark icon, from there you can edit or delete it. If a book exists in several collections (e.g. the same biblical book in different translations) the bookmark will be shown for all, not only for the collection where you added it.

In the settings you can configure the appearance and other details. Note that some settings are only supported for books with the respective metadata.

2. Importing Collections

To use a collection you have to import it first. Either you select the file in the settings (on Firefox OS this requires a file picker app) or you select it in the file manager (only in Firefox OS). After the import is complete you no longer need to keep the file.

The app supports four different formats:

  • Annotated HTML
  • Sword
  • Zefania XML
  • RoundtripHTML

All these formats allow texts with the best quality, but of course they also allow bad quality. The quality of a bible will depend very much on the quality of its source, which may vary from completely broken to excellent. So if you encounter a badly formatted bible, do not blame the format for it. If the errors aren’t that severe, you can export it, edit it by hand, and re-import it.

Annotated HTML

Annotated HTML is the internal format. You can read about it in a separate documentation.

Files are expected to have the extension .html. Alternatively, you can compress them to a ZIP file with just this one file as content.

Sword

Bibles in the Sword format are partially supported. Actually only a small portion of the Sword format is implemented, but the most commonly used parts are. Sword files are expected to be ZIP files with extension .zip. Many files of that format are available at http://www.crosswire.org/sword/modules/ModDisp.jsp?modType=Bibles.

Zefania XML

Bibles in Zefania XML format are supported. Files are expected to have the extension .xml. As with Annotated HTML you may compress them into a ZIP file. Many files of that format are available at https://app.box.com/s/et4h5qhkcf2itcp8nv22 and https://sourceforge.net/projects/zefania-sharp/files/Bibles/.

RoundtripHTML

Bibles in RoundtripHTML format are supported. For documentation on this format see https://github.com/schierlm/BibleMultiConverter. Files are expected to be ZIP files with the extension .zip. Some files of that format are available at https://bibel.github.io/.

3. Future plans

I have several ideas for future improvements.

Improved search

Currently the search has some minor issues. For example, it should be possible to configure the number of results. It should also be possible to highlight the search term in the text.

Different versification schemes

All bibles should internally use the same versification scheme. Currently they just use what the source uses. In almost every case this shouldn’t be a problem, but it might be improved in future. This should also include solutions to handle books like Daniel, which sometimes is split into two books.

Improved comparing translations

When comparing translations, some texts outside verses might get lost, and other minor problems might occur. This should be improved in future.

Other ideas

  • Allow links to books in other collections
  • Support for reading plans
  • A function to select a random chapter or verse
  • Better support for grammer annotations
  • Encode footnotes in a way that makes them usable in plain HTML
  • Improved support for CSS styles: style sheets in collections, arbitrary user style sheets
  • Allow direct editing of books (to correct small mistakes, etc.)

Annotated HTML

1. Basic Concepts

The Annotated HTML format is built on the following concepts:

Collection
Collection of several (or just one) books.
Examples: A bible translation, all encyclicals by John Paul II.
Group
A group inside a collection, containing some of its books or subgroups.
Examples: The New Testament, the Gospels.
Book
A book inside a collection.
Examples: A biblical book, an encyclical.
Section
A part in a book that should appear in the table of contents.
Example: A chapter of a biblical book.
Verse
The smallest part in a book that can be referenced.
Examples: A verse of a biblical book, a paragraph of an encyclical.
Footnote
A footnote in a book.
Examples: An explanation in a biblical book, a list of related texts.
Grammer annotation
A word or a phrase with additional data about its grammer.
Example: A word with a Strong number.
Reference
A reference to a verse of the same book or a verse of a book in the same collection.
Examples: A reference to a related text inside a footnote, a link to a different paragraph in the text.
Other content
Content not inside a verse.
Example: A headline.

These elements can have the following properties:

Collection
Title
The title of the collection.
Example: Full title of the bible translation.
Type
The type of the collection, used for ordering.
Example: bible.
Abbreviation
A short identifier for the collection.
Example: Common abbreviation for the translation. It is suggested to add the language code in square brackets.
Description
An optional short description of the collection.
Example: Name of the translator and date of the translation.
Language
The (main) language of the collection.
Sortkey
An optional key used for the order of collections.
Content
The list of groups and books in the collection.
Group
Title
The name of the group.
Content
The list of groups and books in the group.
Book
Title
The title of the book.
ID
A unique id for the book. This must only contain lower-case letters and numbers. The same book in different collections must use the same id.
Abbreviation
A short identifier for the book. There are three variants: The abbreviation for the book alone, the abbreviation for a section (which should contain %s as placeholder) and the abbreviation for a verse (which should contain %v and can contain %s as placeholder).
Description
An optional short description of the book.
Language
The (main) language of the book, if different from the collection.
Content
The sections of the book.
Section
Abbreviation
A short identifier for the section, which will be inserted into the abbreviation of the book.
Content
The content of the section, consisting of verses, footnotes, references and other content.
Verse
ID
A unique id for the verse. This must start with the ID of the book followed by a hyphen. The same verse in different collections must use the same id.
Abbreviation
An optional abbreviation for the verse. Only necessary if the verse doesn’t contain a visual representation of the ID.
Content
The content of the verse, consisting of verse, footnotes, references and other content. This should contain a visual representation of the ID, and can consist of several parts (e.g. when divided by paragraph breaks). Note that the ID need not match the section, nor the visual representation, e.g. for the psalms which may use different numberings for internal IDs and what is actually shown. Verses may be empty (if it doesn’t exist in a translation), or include other verses (if two verses are translated as union).
Footnote
Name
The footnote mark.
Content
The content of the footnote, consisting of references and other content.
Grammer annotation
Annotation
The grammer annotation.
Content
The annotated word or phrase.
Reference
Link
The ID of the referenced verse or book.
Content
The visible content of the reference, consisting of other content.
Other content
Content
The actual content.

2. HTML representation

The elements and their properties are represented as an HTML document. In most cases each element is represented by an HTML tag with the properties as attributes.

Collection
HTML document, with attribute lang, <title> in the head, and on <body> attributes data-type, data-abbr, data-sort, data-desc, and the content inside it. Feel free to add other meta-data to the head, though it will be ignored and won’t survive export.
Group
<div> with attribute title, and content inside it.
Book
<article> with attributes title, id, data-abbr, data-abbr-section, data-abbr-verse, data-desc, lang, and the content inside it.
Section
<section> with attribute data-abbr, and the content inside it.
Verse
Any HTML element (<span> or <div> if nothing more specific matches), with attribute id or attribute data-idcont (for split verse), and the content inside it. The visual representation of the ID should be enclosed in any HTML element (<span> if nothing more specific matches) with class id. If this doesn’t exist (or if it is different from what you want), use the data-abbr attribute on the verse.
Footnote
Any HTML element (<span> if nothing more specific matches), with attribute data-content and class fn, and the name inside it. Use additional classes to specify the type of the footnote.
Grammer annotation
Any HTML element (<span> if nothing more specific matches), with attribute data-grammer and the content inside it. The annotation should be a Strong number, prefixed by H or G (for Hebrew/Greek) or a RMAC string. You can also add several annotations separated by spaces.
Reference
<a> with attribute href (with #id as content), and the content inside it. If you want to reference more than one verse (especially a range), then include all referenced IDs (including the one in href) separated by spaces (but without #) in the data-ref attribute. You may want to add the class preview to the link, to allow viewing the content like a footnote instead of navigating.
Other content
Any HTML element. Note that images, videos etc. must be embedded as data:-URIs, and external links must add target="_blank". No id attributes are allowed (that would turn them into verses), neither are <article> or <section> nor anything that isn’t allowed according the HTML spec. Avoid inline styles, if you really need them then make sure that they work independently from font size, font color and background color. Headlines should use <h1> for the main title, <h2> for the section title and <h3> to <h6> for subheadlines inside the section.

Some classes are reserved for internal usage. These are body, popup, search-results, button-list, button-list-inline, bookmark, bookmark-add, compare, and any class starting with highlight-.

For the classes used for footnotes and references see the next sections.

Footnotes

Footnotes use the class fn, the content of the footnote is placed in the data-content attribute. So a simple footnote can be added with the following code: <sup class="fn" data-content="Footnote content">*</sup> The result is this footnote.*

Footnotes with additional information that most readers don’t care about should additionally add the class info, which will hide the footnote by default.*

You can also show the footnote inline by adding the class inline.* This can be combined with info.*

References

References are links with a local href, referring to a verse in the same collection, e.g. <a href="#foo-1">. You can also refer to more than one verse by using the data-ref attribute: <a href="#foo-1" data-ref="foo-1 foo-2">

Two additional classes exist for references: The class fn-ref marks a reference that should be treated like a footnote (i.e., an inline reference that should be hidden when other footnotes are hidden, too), the class preview allows showing a preview in a footnote instead of navigating to the book.

Grammer annotations

Currently grammer annotations are only poorly supported. The format is <span data-grammer="G001">word</span>, this shows as: word

Images and other media

You can add images and other media, but you must embed them as data:-URIs. Unless you want an image to show inline, put it into a <figure> tag. Use the class border to add a border.

Book and verse IDs

For biblical books (including Apocrypha), use the OSIS ID, converted to lowercase, which itself is taken from the SBL MoS, removing spaces. For other books, use the following IDs:

BookID
Introduction to Bibleintro
Introduction to OTot
Introduction to NTnt
Appendix to Bibleappendix

For encyclicals and similar texts, use the initial letters of incipit (which are commonly used as abbreviation), converted to lowercase, followed by the year of publishing, e.g. Laudato Si’ should have the ID ls2015.

For verse IDs for biblical books use bookid-chapter-verse (the versification scheme to use is not yet fixed), for other books use just bookid-verse.

Collection types and sortkeys

For the type, use bible for bibles (including any collection of biblical books), and doc for any collection of documents by the pope, bishops, etc., including collection of encyclicals, apostolic exhorations, apostolic constitutions, etc. For internal use there are help for the help and test for tests.

Usually, no sortkey is neccessary. For documents that should appear in chronological order, use the year of the first document in the collection as sortkey.

4. Code examples

The basic document structure should be like this:

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
<title>Title</title>
</head>
<body data-type="bible" data-abbr="Foo [en]">

</body>
</html>

Each book should have a basic structure like this:

<article id="foo" title="Foo" data-abbr="FOO" data-abbr-section="FOO %s" data-abbr-verse="FOO %s:%v">
<section data-abbr="1">

</section>
</article>

Sections and verses in a normal biblical book should be encoded like this:

<section data-abbr="1">
<p><span class="id-big">1&nbsp;</span><span id="foo-1-1"><span class="id">1&nbsp;</span>First verse</span>
<span id="foo-1-2"><span class="id">2&nbsp;</span>Second verse</span></p>
</section>

For encyclicals the usual structure should be like this:

<section data-abbr="1">
<h2>Headline</h2>
<div id="foo-1"><p><span class="id">1.&nbsp;</span>First paragraph</p>
</div>
<div id="foo-2"><p><span class="id">2.&nbsp;</span>Second paragraph</p>
<p>More second paragraph</p>
</div>
</section>

Sometimes verses aren’t as easy as that. If you need to split a verse, use a structure like this:

<p><span id="foo-1-1"><span class="id">1&nbsp;</span>First verse, part 1</span></p>
<h2>Headline after paragraph break</h2>
<p><span data-idcont="foo-1-1">First verse, part 2</span></p>

When two verses are merged into one, use a structure like this:

<span id="foo-1-1"><span id="foo-1-2"><span class="id">1/2&nbsp;</span>Verse</span></span>

For missing verses, you can add an empty verse (which will allow bookmarks for that verse to show if you added one from another version of that book where the verse is not empty). There is no need that the ID of the verse and the label agree. It is totally fine to have an ID foo-1-1 in the section number 2, and showing <span class="id">3</span>.
